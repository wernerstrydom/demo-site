EOL status and risk

Sophos declared the SG series end of life in 2023. No further firmware updates are issued by Sophos. The original Sophos OS (SFOS) is not used in this lab. Both units run Ubuntu Server 24.04 LTS on the bare hardware, which receives standard Ubuntu security updates.

The hardware itself is not EOL in the meaningful sense: the Intel Atom C2358 and the network interfaces are well-supported by the Linux kernel. The mSATA SSD is replaceable. The platform will receive OS security updates for the lifetime of Ubuntu 24.04 LTS (April 2029 standard support, April 2034 extended).

Risk assessment: Running Ubuntu on this hardware does not inherit Sophos's EOL status. The risk is hardware failure, not software vulnerability. The Intel Atom C2358 is a known-stable platform with no significant errata affecting this use case.

Why this hardware

The Sophos SG 210v3 is available on the secondary market for $30–80 USD (2024 prices). For that price, you get a 1U rack appliance with 6 GbE ports, 2 SFP+ ports, ECC memory, and a serial console. The Intel i210 NICs are among the best-supported GbE controllers in Linux. The SFP+ ports use the Intel 82599, which supports SR-IOV and is well-supported in the kernel.

The alternative for a comparable feature set is a purpose-built x86 firewall appliance at $300+, or a consumer router that lacks ECC memory, a serial console, and multiple GbE ports. The Sophos hardware is the better value for a lab environment where the goal is learning, not vendor support.

The trade-off is power consumption: the Atom C2358 draws more power than a modern low-power SoC (e.g., Intel N100). For a lab that runs 24/7, this is a meaningful cost. For a lab that is occasionally powered down, it is not.

Installing Ubuntu

The SG 210v3 boots from the mSATA SSD. The BIOS is a standard AMI UEFI implementation. Ubuntu 24.04 LTS installs without modification. The serial console is required for installation if no display is connected.

BIOS settings to change before installation:

  • Set boot order: mSATA SSD first.
  • Disable Sophos-specific boot entries if present.
  • Enable legacy serial console redirection (115200 8N1) for headless operation.

Ubuntu installer serial console: append console=ttyS0,115200n8 to the kernel command line in the GRUB boot menu during installation. After installation, add the same parameter to /etc/default/grub and run update-grub.

GRUB_CMDLINE_LINUX="console=ttyS0,115200n8"

Network interface mapping

The interface names assigned by the kernel depend on the firmware and driver version. Use ip link to enumerate interfaces. The mapping to physical ports is determined by PCI bus address. The following is typical but should be verified on each unit:

Kernel name Physical port Controller
enp1s0 – enp1s5 GbE ports 1–6 (front panel) Intel i210
enp2s0f0, enp2s0f1 SFP+ ports 1–2 (rear) Intel 82599

Persistent interface naming via systemd-networkd or udev rules is recommended. Do not rely on kernel-assigned names across reboots without pinning.
