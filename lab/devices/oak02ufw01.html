Role and rationale

oak02ufw01 is the primary firewall for the lab. It handles inter-VLAN routing, stateful packet filtering, and NAT for outbound internet access. It is the VRRP master for all VLAN gateway virtual IPs.

The choice of Ubuntu on bare metal rather than a dedicated firewall distribution (pfSense, OPNsense, VyOS) is deliberate. Ubuntu provides a well-understood base, standard tooling, and a long support lifecycle. The firewall rules are managed with nftables. The VRRP failover is managed with keepalived. Connection state synchronisation uses conntrackd. Each component is a standard Linux tool with extensive documentation.

The trade-off: a dedicated firewall distribution provides a GUI and pre-configured defaults. Ubuntu requires manual configuration of each component. For a lab environment where the goal is understanding, manual configuration is the point. For a production environment where operational simplicity matters, a dedicated distribution may be the better choice.

VRRP configuration

keepalived manages VRRP. oak02ufw01 is the master with priority 100. oak03ufw01 is the backup with priority 90. Failover occurs within 3 seconds of master failure (3 × advert_int of 1 second).

vrrp_instance VI_MGMT {
    state MASTER
    interface enp1s0
    virtual_router_id 10
    priority 100
    advert_int 1
    virtual_ipaddress {
        10.0.10.1/24
    }
}

vrrp_instance VI_PROD {
    state MASTER
    interface enp1s0.20
    virtual_router_id 20
    priority 100
    advert_int 1
    virtual_ipaddress {
        10.0.20.1/24
    }
}

Additional VRRP instances follow the same pattern for VLANs 30, 40, and 50.

Connection state synchronisation

conntrackd synchronises the netfilter connection tracking table between oak02ufw01 and oak03ufw01 over the dedicated sync link (enp1s1). When failover occurs, established connections continue without interruption on the new master.

Configuration at /etc/conntrackd/conntrackd.conf. State is exchanged over UDP multicast on the sync network (192.168.100.0/30). oak02ufw01 uses 192.168.100.1; oak03ufw01 uses 192.168.100.2.

Firewall rules

Firewall rules are managed with nftables. The ruleset is version-controlled in the internal git server. Rules are applied via a systemd service that loads the ruleset at boot.

The policy is default-deny for inter-VLAN traffic. Permitted flows are explicitly allowed. Internet access for the Production VLAN is permitted via NAT masquerade. The IoT VLAN has internet access only; no inter-VLAN access.

Maintenance notes

Before performing maintenance on oak02ufw01, verify that oak03ufw01 is operational and that VRRP failover works. Procedure:

  1. SSH to oak03ufw01. Verify keepalived is running: systemctl status keepalived.
  2. On oak02ufw01, stop keepalived: systemctl stop keepalived. oak03ufw01 should assume the VIPs within 3 seconds.
  3. Verify failover: ping 10.0.20.1 from a Production VLAN device. Verify connectivity continues.
  4. Perform maintenance on oak02ufw01.
  5. Restart keepalived on oak02ufw01: systemctl start keepalived. oak02ufw01 will preempt oak03ufw01 and resume as master.
Warning: Do not perform maintenance on both firewalls simultaneously. If both are unavailable, all inter-VLAN routing and internet access fails. There is no third node.
