Lab › Networking

The lab network is segmented into VLANs by trust level and function. Each VLAN has a defined subnet, default gateway, and firewall policy. Inter-VLAN routing is handled by the firewall pair.

VLAN assignments

VLAN ID Name Subnet Default gateway Purpose
10 Management 10.0.10.0/24 10.0.10.1 Out-of-band management. Servers, switches, firewall management interfaces.
20 Production 10.0.20.0/24 10.0.20.1 Primary network. Trusted devices, daily-use machines.
30 Lab 10.0.30.0/24 10.0.30.1 Isolated experiment network. VMs, containers, test services.
40 Vintage 10.0.40.0/24 10.0.40.1 Legacy hardware (PowerPC, MIPS, SPARC). Restricted outbound access.
50 IoT 10.0.50.0/24 10.0.50.1 IoT devices. No inbound access from other VLANs. Internet access only.

Firewall policy summary

Source VLAN Destination VLAN Policy Notes
Management All Allow (restricted) SSH, SNMP, HTTPS only. No general traffic.
Production Management Deny Production devices cannot reach management interfaces.
Production Lab Allow (restricted) Specific ports only. Reviewed per-service.
Lab Production Deny Lab traffic cannot reach production by default.
Vintage Production Deny Vintage machines are untrusted. No access to production.
IoT Any internal Deny IoT devices have internet access only.

Firewall HA

The firewall pair (oak02ufw01 and oak03ufw01) uses VRRP for gateway failover. Each VLAN has a virtual IP (VIP) that floats between the two nodes. keepalived manages VRRP. conntrackd synchronises connection state.

The VRRP virtual IPs are the .1 addresses in each subnet (e.g., 10.0.20.1 for the Production VLAN). The physical management IPs for the firewall nodes are 10.0.10.21 (oak02ufw01) and 10.0.10.22 (oak03ufw01).

Trade-off: VRRP failover eliminates single-node downtime but introduces split-brain risk. See High Availability is a Trade-off, Not a Feature for a full analysis.

DNS

Internal DNS is served by a resolver on the Management VLAN. The domain lab.internal is used for internal hostnames. All devices have A records. PTR records are maintained for all static IPs.

Upstream DNS uses a resolver with filtering (Pi-hole). The filtering list is reviewed quarterly. Filtering is applied to the Production and IoT VLANs. The Lab and Vintage VLANs use unfiltered upstream DNS to avoid interference with experiments.
