The lab is a collection of physical hardware running a mix of architectures, operating systems, and network configurations. It exists to learn, to experiment, and to maintain skills that cloud environments do not require. Documentation is kept precise and current.

Lab summary

The lab spans five hardware architectures: x86-64, PowerPC, MIPS, SPARC, and ARM. The x86-64 machines handle production workloads — virtualisation, storage, and network services. The vintage architectures (PowerPC, MIPS, SPARC) are used for cross-platform testing and to maintain familiarity with non-x86 systems. The ARM cluster (Raspberry Pi 4) provides a low-power environment for distributed systems experiments.

Network segmentation is enforced via VLANs. Each segment has a defined trust level and firewall policy. The firewall pair (oak02ufw01 and oak03ufw01) runs Ubuntu on repurposed Sophos SG 210v3 appliances. High availability is configured via VRRP (keepalived) and connection state synchronisation (conntrackd).

Note: All IP addresses documented here are RFC 1918 private addresses. No public IP addresses are documented on this site.
