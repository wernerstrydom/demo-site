Network

Topology

The network uses VLANs for segmentation. A managed switch handles VLAN tagging. Ubuntu-based firewalls (oak02ufw01, oak03ufw01) provide routing, firewall, and VPN.

VLAN Subnet Purpose
10 10.0.10.0/24 Management (IPMI, iDRAC)
20 10.0.20.0/24 Production workloads
30 10.0.30.0/24 Lab and testing
40 10.0.40.0/24 Vintage systems
50 10.0.50.0/24 IoT devices

Firewall rules restrict inter-VLAN traffic. Management VLAN is isolated. IoT VLAN has no inbound access from other VLANs. Lab VLAN can initiate connections but receives no unsolicited traffic.

Services

DNS: Unbound with local zone for internal names. Forwards to upstream resolvers over TLS. Blocks ad domains.

DHCP: ISC DHCP with static reservations. Network boot support via PXE. Option 66 points to TFTP server.

Monitoring: Prometheus scrapes exporters on each system. Grafana visualizes metrics. Alertmanager sends notifications.

Storage: ZFS on Server 1 provides network storage via NFS and SMB. Snapshots run hourly. Replication to external drive runs daily.

VPN: WireGuard for remote access. Split tunnel configuration. DNS pushed to clients.

Projects

Cross-Architecture Build System

A Docker-based build system compiles software for x86-64, ARM, PowerPC, and MIPS. It uses QEMU for emulation and cross-compilation toolchains. The system builds packages for vintage systems that no longer have official support.

The build process runs in containers to ensure reproducibility. Each architecture has a dedicated container image with the appropriate toolchain. The Makefile orchestrates builds across architectures.

Automated Testing Pipeline

A CI/CD pipeline tests website compatibility across browsers and architectures. Virtual machines run period-appropriate browsers. Selenium captures screenshots. A comparison tool detects visual regressions.

The pipeline runs on git push. It tests on Netscape Navigator 4, Internet Explorer 5, Mozilla 1.0, and modern browsers. The results show whether changes broke compatibility.

Infrastructure Monitoring

A custom monitoring system collects metrics from diverse systems. It supports SNMP for network equipment, SSH for Unix systems, and HTTP for modern services. The collector runs on Server 2.

The system handles systems that can't run modern monitoring agents. It uses standard protocols available on vintage hardware. Data goes to Prometheus for storage and querying.

Configuration

Configuration management uses Ansible. Playbooks define system state. The inventory groups systems by function. Variables handle environment-specific values.

The approach is minimal. Ansible ensures packages are installed, services are running, and configuration files are present. It doesn't manage every file. Manual configuration is documented in runbooks.

Testing happens before deployment. A staging environment mirrors production. Changes apply to staging first. Validation includes service checks and integration tests. Production deployment happens after validation passes.
