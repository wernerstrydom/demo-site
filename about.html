About Me

I'm a software developer and technology enthusiast with a passion for web standards, open source software, and computing history. I believe in building things that last, respecting user freedom, and maintaining the open and accessible nature of the web.

My interest in computing spans from vintage systems to modern technologies. I maintain a home lab with diverse hardware including PowerPC, MIPS, and SPARC systems alongside modern x86 and ARM machines. This hands-on experience with different architectures has taught me the value of standards, portability, and thoughtful design.

Background

I've been involved with web development since the late 1990s, starting in the era of table-based layouts and browser wars. I've seen the web evolve from simple HTML documents to complex applications, and I've learned to appreciate both the simplicity of the early web and the capabilities of modern browsers.

Throughout my career, I've worked on projects ranging from small personal websites to large-scale web applications. I've learned that the best solution depends on the specific requirements, and that sometimes the simplest approach is the most appropriate.

Philosophy

I believe in several core principles:

  • Standards over proprietary solutions: Open standards enable interoperability and longevity
  • Accessibility for all users: Technology should be inclusive and work for everyone
  • Simplicity when possible: Complex solutions should be reserved for complex problems

About This Website

Why Progressive Enhancement?

This website is built using progressive enhancement as both a technical approach and a philosophical statement. In an era where many websites require specific browsers, fast connections, and powerful hardware, I wanted to create something that works for everyone.

Progressive enhancement starts with a foundation of semantic HTML that works in any browser, then adds layers of CSS and JavaScript to enhance the experience for browsers that support them. This means the site works on a 1999-era browser while still providing a modern, polished experience on current browsers.

Technical Implementation

The website uses HTML 4.01 Transitional as its base markup language. This version of HTML has excellent support across all browsers from the late 1990s onwards. The markup is semantic, using appropriate elements for headings, paragraphs, lists, and other content structures.

CSS is applied through an external stylesheet that uses progressive enhancement techniques. Basic styles that work in older browsers provide readable typography and layout. Modern browsers receive additional enhancements through CSS features like Grid, Flexbox, custom properties, and media queries. These advanced features are applied using @supports and other feature detection methods, ensuring they don't break the experience in older browsers.

JavaScript is used minimally and unobtrusively. The site is fully functional without JavaScript, but browsers that support it receive small enhancements like smooth scrolling and theme switching. All JavaScript uses feature detection to ensure compatibility.

Performance

The website is designed to be lightweight and fast-loading:

  • Total page size: Less than 50KB including HTML, CSS, and JavaScript
  • No external dependencies or frameworks
  • No tracking scripts or analytics
  • No web fonts (system fonts only for maximum compatibility)
  • Minimal HTTP requests

This ensures the site loads quickly even on slow connections or older hardware.

Source Code

The website's source code is intentionally simple and readable. Feel free to view the source in your browser to see how it's built. There are no build steps, minification, or obfuscation. What you see is what there is.

The code follows web standards and best practices from both the HTML 4 era and modern development. Comments in the code explain the progressive enhancement techniques used.
